# 识别全部号码状态 & 提升准确率

本文说明如何识别 da2 完整状态表(id 2-20,见 [`INTEGRATION.md` §3.7](./INTEGRATION.md))并持续提升准确率。

## 一、识别原理回顾

号码状态识别 = **分析接通前早期媒体的声音**:

- **信号音**(回铃/忙音/拥塞/彩铃/静音):本地 DSP(频率 + 节奏)直接判定。
- **语音提示音**(关机/空号/停机/通话中/语音信箱…):声音特征与**样本库**比对(音频指纹)为主,**ASR 兜底**为辅。

识别为某状态 = "系统呼叫该号码听到的声音与该状态样本匹配上了",不等于号码客观上一定是该状态(线路/运营商接续问题也可能放错提示音)。这是方法本身的边界。

## 二、识别出全部 19 个状态的两条路径

### 路径 A:样本库(指纹)—— 主力,快且省

为每个状态收录其提示音样本(可多条变体),靠指纹匹配命中。覆盖度 = 准确率。

- 用 `sampletool states` 查看需覆盖的 19 个状态;
- 每个状态尽量收录**多个运营商 / 多地区 / 多措辞**的变体样本(同一 `category/alias`,不同 `name`);
- 入库会按标准表归一 `category/alias`(给出其一即补全另一,并写入 `id`)。

### 路径 B:ASR 兜底 —— 覆盖样本库尚未收录的措辞

样本库未命中(`prompt`)时,转写文本按关键词归类到 19 个状态之一(规则见 `states.py` 的 `keywords`)。开启 `--asr --asr-autolearn` 后,ASR 归类的段会**自动补入样本库**,下次走指纹快路径——系统越用越准。

> 关键词规则可在 `states.py` 给各状态补充更多措辞(如不同运营商的话术),即时生效、无需训练。

## 三、提升准确率的具体手段(已内置)

| 手段 | 机制 | 开关/位置 |
|---|---|---|
| **多样本/状态** | 同状态收录多变体,最近邻取最高分 | 样本库(多条 `name`,同 `alias`) |
| **置信度分级** | `ACCURACY`/`INACCURACY`/`LOOSE`,仅 `ACCURACY` 触发挂机/上报 | `--accuracy` / `--inaccuracy` |
| **指纹+ASR 交叉校验** | 指纹候选为 `INACCURACY` 时用 ASR 复核,一致则升 `ACCURACY`(`confirmed_by=asr`) | 同时配置样本库 + `--asr` |
| **ASR 兜底 + 自学习** | 未命中转写归类,并自动回流补库 | `--asr --asr-autolearn` |
| **未命中回流** | `prompt` 段落盘,人工 `sampletool promote` 打标入库 | `--capture-dir` + `sampletool` |
| **指纹鲁棒性** | 电话频带对数频带 + 时间平滑 + 逐帧去均值 + 时间归一,抗增益/噪声 | `fingerprint.py` |

## 四、推荐落地流程(从 0 到高准确率)

1. **跑起来**:先用本地 DSP 拿到回铃/忙音/拥塞;识别服务先空库 + `--capture-dir`。
2. **采集**:真实外呼一段时间,未命中的提示音自动回流到 capture 目录(也可 `recordpath` 录全段)。
3. **打标入库**:`sampletool pending` 看待标注 → 试听 → `sampletool promote` 按 19 状态标注入库。优先覆盖高频状态(关机/空号/停机/通话中,da2 称其覆盖 >99%)。
4. **开 ASR 兜底**:接入真实 ASR(`asr.create_asr` 里实现 Whisper/云端),`--asr-autolearn` 让系统自学习扩库。
5. **持续运营**:定期审阅回流样本、补关键词规则、给易混状态加更多变体样本;用 `--accuracy` 调节灵敏度(高阈值更稳、低阈值更全)。

## 五、调参建议

- **追求"准"**(宁可漏判):调高 `--accuracy`(如 0.85+),依赖交叉校验升级;只在 `ACCURACY` 时挂机。
- **追求"全"**(尽量识别):调低 `--inaccuracy`,开 ASR 兜底,容忍 `INACCURACY` 仅作提示不挂机。
- **降低误判**:对同一通话要求连续 2 次一致结果再采信(提示音通常循环播放);把"稍后再拨"类泛化提示在 `states.py` 里降优先级(已是最末)。

## 六、边界与排查

- 识别为空号 ≈ 99.999% 确定"听到的是空号提示音",但不等于号码客观为空号(线路/运营商问题)。排查:开录音,听未接通前声音,区分线路问题 vs 样本库问题。
- 关机/空号/停机/通话中:提示音相对标准,样本库覆盖 >99% 即高准确。
- 语音信箱/留言(来电提醒类):措辞多样,需更多样本 + ASR 兜底,覆盖率目标 >90%。
- 彩铃/真人:DSP 先粗分 `other`;精确区分需音乐/语音判别(可在服务端扩展)。
